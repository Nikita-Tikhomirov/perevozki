# Telegram-уведомления о заявках Perewozki.by

## Цель

Дублировать новые заявки со всех форм Perewozki.by в Telegram клиента, сохранив
действующую отправку по email и запись заявок в FormIt.

## Архитектура

- Отдельный сервер и постоянно работающий процесс не используются.
- После успешной проверки и сохранения формы MODX запускает серверный FormIt-hook.
- Hook отправляет HTTPS-запрос в Telegram Bot API с текущего хостинга сайта.
- Токен бота и идентификатор получателя хранятся в закрытых системных настройках
  MODX и не попадают в шаблоны, репозиторий или клиентский JavaScript.
- Клиент один раз открывает ссылку созданного бота и нажимает `/start`. После
  этого сайт получает числовой chat ID и может доставлять уведомления.

## Состав уведомления

Сообщение содержит:

- название и URL страницы, с которой пришла заявка;
- имя;
- телефон;
- email, если поле присутствует;
- текст сообщения или параметры заявки;
- дату и время получения.

HTML из пользовательских полей экранируется. Пустые поля не выводятся.

## Email-шаблон

- Существующая SMTP-доставка и адрес получателя сохраняются.
- Вместо стандартного дампа всех FormIt-полей используется один HTML-шаблон в
  цветах Perewozki.by.
- В письме показываются только название страницы, URL, имя, телефон, email,
  сообщение и время получения заявки.
- `g-recaptcha-response`, `savedForm.*`, IP-адрес и служебные поля FormIt в
  письмо не выводятся.
- Тема письма: `Новая заявка с Perewozki.by — <телефон>`.
- Для почтовых клиентов без HTML добавляется короткое текстовое представление.

## Поведение при ошибках

- Email и сохранение в FormIt остаются основными каналами и не зависят от
  доступности Telegram.
- Telegram-hook выполняет короткий запрос с таймаутом и одной повторной попыткой.
- Если Telegram временно недоступен, заявка не теряется: форма остаётся успешно
  отправленной, а причина ошибки записывается в журнал MODX без токена бота.

## Подключение

- Создать отдельного бота Perewozki.by через BotFather.
- Добавить системные настройки `telegram_bot_token` и `telegram_chat_id`.
- Создать один переиспользуемый snippet-hook `TelegramFormNotify`.
- Создать общий чанк `PerewozkiFormEmail` и указать его в `emailTpl` всех форм.
- Подключить hook ко всем четырём AjaxForm в общем footer-шаблоне.
- Не менять существующие hooks `rcv3`, `FormItSaveForm` и `email`.

## Проверка

1. Отправить заявку с тестовой меткой через опубликованную маршрутную страницу.
2. Проверить успешный ответ формы и запись в FormIt.
3. Проверить получение письма в Mail.ru.
4. Проверить получение Telegram-сообщения клиентом.
5. Убедиться, что в журнале MODX нет ошибок SMTP и Telegram.
