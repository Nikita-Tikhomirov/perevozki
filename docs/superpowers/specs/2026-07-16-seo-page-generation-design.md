# Генерация SEO-страниц для маршрута Минск — Узда

## Цель

Создать и опубликовать на `perewozki.by` 24 тестовые SEO-страницы — по одной для каждого запроса из файла `СЕО для ГЕНЕРАЦИИ.xlsx`. Все страницы используют утверждённый шаблон маршрута Минск — Узда, существующие блоки и стили сайта. Существующие ресурсы MODX не перезаписываются.

## Объём первой генерации

- Один маршрут: Минск — Узда.
- 24 типа услуг из строк 2–25 листа `Лист1`.
- Первый вариант из пяти для title, description и вводного абзаца.
- Публикация доступна по прямой ссылке для проверки, но страницы закрыты от индексации и скрыты из меню.
- Массовая генерация для других городов, включение индексации и замена существующих посадочных страниц в эту задачу не входят.

## Сопоставление Excel

| Столбцы | Назначение |
|---|---|
| A | запрос и H1 |
| B–F | варианты title |
| G–K | варианты description |
| L–P | варианты первого абзаца |
| Q | вводный текст блока цены |
| R | вводный текст блока для частных лиц |
| S | вводный текст блока для бизнеса |
| T | заголовок FAQ |
| U | готовая HTML-разметка FAQ |
| V | заголовок контактного блока |
| W | заголовок фотогалереи |

Плейсхолдеры `{Город1}` и `{Город2}` заменяются только на переданные значения маршрута. Исходные тексты Excel не дописываются и не перефразируются генератором.

## Формирование страницы

Базой служит утверждённый `index.html`. Генератор меняет только SEO-зависимые поля:

- `<title>` и meta description;
- H1 и вводный абзац;
- заголовок и вводный абзац цены;
- заголовки и вводные абзацы блоков для частных лиц и бизнеса;
- маршрутные заголовки существующих секций;
- FAQ из готового HTML столбца U;
- заголовки фотогалереи и контакта;
- заголовок перелинковки по Беларуси.

Карточки услуг, автопарк, преимущества, фотографии, контакты, перелинковка, модальные формы и подключаемые MODX-чанки остаются теми же, что в утверждённом шаблоне. Ссылки карточек продолжают вести на существующие страницы сайта, а отсутствующие направления — на главную.

## URL и защита данных

Тестовые URL получают отдельный префикс:

`seo-2026-<slug-запроса>-minsk-uzda`

Каждый ресурс содержит технический HTML-маркер партии. Перед созданием или обновлением публикация проверяет:

1. существует ли точный alias;
2. если существует — содержит ли ресурс маркер этой партии;
3. если маркера нет — операция прекращается с ошибкой без изменения ресурса.

Ресурсы создаются верхнего уровня с параметрами `published=1`, `hidemenu=1`, `searchable=0`, `cacheable=0`. Используется noindex-шаблон предпросмотра. Секреты передаются только через переменные окружения и не сохраняются в репозитории.

## Публикация и результат

Скрипт сначала выполняет локальную генерацию всех 24 страниц и проверяет уникальность alias. Затем один раз загружает общие ассеты и публикует ресурсы через MODX API. После успешной публикации сохраняется JSON-манифест с запросом, ID ресурса и публичным URL.

## Проверка

- модульные тесты импорта XLSX, плейсхолдеров, slug, HTML-подстановок и защиты от конфликтов;
- проверка количества: ровно 24 строки и 24 уникальных URL;
- HTTP-проверка всех страниц: код 200, `noindex`, правильные title/H1 и отсутствие незаменённых плейсхолдеров;
- браузерная проверка первой, средней и последней страницы на desktop и mobile;
- проверка FAQ, карточек-ссылок, форм и отсутствия горизонтального переполнения;
- финальный запуск полного набора тестов и глобального smoke harness.
