# Perevozki

Одностраничный шаблон маршрутной SEO-страницы для Perewozki.by.

## Что реализовано

- демонстрационная страница «Перевозка грузов Минск – Узда»;
- адаптивный дизайн для desktop, tablet и mobile;
- реальные фотографии автопарка и выполненных работ клиента, включая отдельную большую машину до 5 тонн;
- порядок и тексты блоков из `NEW Новый шаблон для СЕО 2026.docx` и первой строки `СЕО для ГЕНЕРАЦИИ.xlsx`;
- таблица цен из ТЗ, премиальные карточки частных и корпоративных сценариев, этапы заказа, автопарк, преимущества, FAQ, фотографии, заявка и города;
- рабочие ссылки телефона и мессенджеров;
- карточки для частных лиц и бизнеса ведут на существующие тематические страницы Perewozki.by, а при отсутствии подходящего раздела — на главную;
- существующие блоки заявки и услуг подключены штатными MODX-чанками;
- полный каталог содержит 118 уникальных направлений из Минска и обычные ссылки на реальные страницы;
- генератор создаёт 2 832 страницы: 24 запроса Excel × 118 направлений;
- обратные направления пока не создаются; Title, Description и первый абзац независимо и воспроизводимо миксуются из пяти Excel-вариантов;
- публичные страницы используют индексируемый шаблон и маркер владения; прежние тестовые `seo-2026-*` URL остаются `noindex`.

## Фотографии

В `assets/photos/` находятся выбранные исходные фотографии клиента без генеративной замены и цветной тонировки. Файлы `assets/private-services-sprite.jpg` и `assets/business-services-sprite.jpg` содержат созданные специально для премиальных карточек услуг иллюстрации без текста и логотипов.

## Публикация

Тестовая страница: [https://perewozki.by/seo-preview-2026/](https://perewozki.by/seo-preview-2026/).

В MODX создан отдельный скрытый ресурс и отдельный шаблон. Структурные чанки `head`, `header`, `index-menu` и `footer` остаются в шаблоне. Чанк `s-question` вставлен после блока «Как заказать», штатный `s-adv` — после автопарка перед расширенными преимуществами, а `s-services` — после фотографий, точно по порядку ТЗ. Стили и изображения изолированы в `/assets/seo-preview-2026/`, главная страница не изменена.

Для повторной публикации нужны переменные окружения `FTP_HOST`, `FTP_USERNAME`, `FTP_PASSWORD`, `MODX_USERNAME`, `MODX_PASSWORD`, после чего запускается:

```powershell
python scripts\deploy_modx_preview.py
```

## Генерация всех SEO-страниц

Безопасный локальный прогон всех 2 832 страниц не подключается к MODX и сохраняет будущие URL в JSON:

```powershell
python scripts\deploy_modx_generated.py `
  --xlsx "C:\path\to\СЕО для ГЕНЕРАЦИИ.xlsx" `
  --all-routes `
  --dry-run
```

Для публикации используются те же пять переменных окружения, что и для предпросмотра. `--resume` продолжает работу по сохранённому манифесту после сетевого сбоя:

```powershell
python scripts\deploy_modx_generated.py `
  --xlsx "C:\path\to\СЕО для ГЕНЕРАЦИИ.xlsx" `
  --all-routes `
  --resume
```

Все опубликованные страницы проверяются по манифесту:

```powershell
python scripts\verify_seo_routes.py outputs\seo-2026-all-minsk-routes.json --workers 8
```

Повторный запуск обновляет только ресурсы с маркером генерации. До первой записи скрипт проверяет весь набор alias; при совпадении с чужим ресурсом публикация прекращается без его перезаписи.
